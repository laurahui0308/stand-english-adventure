## 一、词库系统

### 1.1 词库规模
- **总计 420 个单词**，分为 3 个难度等级
- 每个等级包含 6 个主题：动物、食物、学校、自然、日常、情感

| 等级 | 名称 | 对标 | 每主题词数 | 总词数 |
|---|---|---|---|---|
| Level 0 | 替身使者 | KET/A2（3-5年级） | 40 | 240 |
| Level 1 | 吸血鬼 | PET/B1（5-7年级） | 15 | 90 |
| Level 2 | 究级生物 | 中考/B1+（7-9年级） | 15 | 90 |

### 1.2 词库结构
- 每个单词存储 `{en, zh}`，干扰选项从同主题中自动随机生成
- 初始化时自动为每个单词生成 4 个选项（含 1 个正确答案 + 3 个干扰项）

### 1.3 默认难度
- 默认使用 **Level 0（替身使者）**
- 首页不展示难度切换入口，用户可在 **个人中心 → 难度等级** 切换
- 难度切换影响：战斗模式、每日挑战、无限模式的出题范围

## 二、智能复习系统

### 2.1 单词状态
每个单词有三种状态，存储在 `state.wordHistory` 中：

| 状态 | 含义 | 行为 |
|---|---|---|
| `new` | 未学过 | 作为新词候选 |
| `review` | 答错待复习 | 次日优先出现 |
| `mastered` | 已掌握 | 不再出现 |

### 2.2 每日挑战词池
- **每日 5 题**，每新的一天自动构建词池
- 词池优先级：`review` 单词（昨天答错的）→ `new` 单词（未学过的）
- 答对的单词标记为 `mastered`，不再出现
- 答错的单词标记为 `review`，第二天重新出现
- 复习单词在界面上显示 🔁 标识和琥珀色高亮

### 2.3 战斗模式 & 无限模式
- `getRandomWord()` 优先选取 `new` 单词
- 30% 概率随机插入一个 `review` 单词（巩固复习）
- 跳过所有 `mastered` 单词
- 答题后自动记录对错到 `wordHistory`

## 三、阅读理解系统

### 3.1 RAZ Level L 阅读篇目
- **共 20 篇**，每篇约 300-400 词
- 分为两批（每周 10 篇），题材涵盖科学、自然、历史、小说、传记、健康等

### 3.2 每周轮换机制
- **每天展示一篇**，不重复
- 篇目按周批次解锁：
  - 第 1 周（6月1日起）：r1 ~ r10
  - 第 2 周（6月8日起）：r11 ~ r20
- 每 **周一 8:00 AM** 自动解锁下一批
- 未解锁篇目显示 🔒 和解锁日期
- **读完不再循环**：当周全部完成后显示"🏆 本周阅读全部完成"
- 下一篇目将在下周一解锁

### 3.3 今日阅读入口
- 首页"今日阅读"点击后**直接进入当天推荐文章详情页**
- 阅读试炼入口（副命令区）进入全部篇目列表

## 四、发音朗读功能

### 4.1 答题后自动朗读
- 无论答案对错，选择后自动朗读：
  1. 英文单词标准发音（`en-US`）
  2. 900ms 后朗读中文翻译（`zh-CN`）
- 适用模式：战斗模式、每日挑战、无限模式
- 通过 Web Speech API 实现

### 4.2 Toast 提示增强
- 正确答案 Toast 显示：`✅ +1⭐ | apple = 苹果`
- 错误答案 Toast 显示：`❌ 正确答案: 苹果 (apple) 🔁明天复习`

## 五、多用户系统

### 5.1 用户识别
- 基于**名称**区分用户，无需密码
- 首次打开弹出创建页面，输入名字后进入
- 每个用户数据独立存储，格式：`standEnglishAdventure_<用户名>`

### 5.2 用户弹窗
- 点击首页大卡片区中的用户名称（原 `DORMANT` 位置）→ 弹出用户信息窗口
- 弹窗功能：
  - ✕ 关闭按钮
  - 显示当前用户名（只读）
  - ✏️ 编辑名称：切换为编辑模式 → 输入新名称 → 保存
  - 修改名称时自动迁移 localStorage 数据到新 key

### 5.3 个人中心用户管理
- "用户管理"区域列出所有用户
- 当前用户显示 ⭐ 标识 + ✎ 编辑按钮
- 仅可编辑名称，无删除和切换入口
- 切换用户通过首页弹窗中的已有用户列表完成

### 5.4 旧数据迁移
- 检测到旧版单 key 数据时，创建新用户后自动迁移

## 九、数据存储

### 9.1 存储结构
```
localStorage:
  standEnglishAdventure_users   → ["小明", "小红"]
  standEnglishAdventure_active  → "小明"
  standEnglishAdventure_小明    → { stars, bossDone, wordHistory, ... }
  standEnglishAdventure_小红    → { stars, bossDone, wordHistory, ... }
```

### 9.2 存储内容
- 游戏进度（星星、能量、Boss、成就）
- 单词学习历史（每个单词的 mastered/review 状态）
- 每日打卡（日期、连续天数、进度）
- 阅读进度（已完成篇目、当前队列、周批次）
- 对话进度
- 用户设置（难度等级）
