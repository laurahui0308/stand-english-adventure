## 2. 产品概述与背景

### 2.1 市场背景

**现状：**
- 国内中小学生英语学习的刚性需求长期存在
- 现有英语学习APP的两大通病：
  - **"学习感"过重**：界面设计像教科书，孩子打开就有抵触心理
  - **反馈周期过长**：背了100个词才能"看到效果"，孩子等不了那么久
- 移动端游戏化学习产品（如多邻国、百词斩）虽然开创了先河，但本质上仍是"学习工具"而非"游戏"
- JOJO系列在B站累计播放量超30亿，10-15岁群体渗透率高

**机会：**
> 将JOJO风格的美术和战斗系统与英语学习结合，让孩子以为自己在玩JOJO版宝可梦，实际上在学英语。

### 2.2 产品定位

> 一款以**替身收集 × 英语学习**为核心的HTML5轻游戏，面向10-15岁中小学生，覆盖单词、阅读、口语训练，通过替身进化、地图探索、每日打卡构建持续学习动力。

### 2.3 产品原则（4条铁律）

1. **游戏优先，学习隐形** — 孩子打开时看到的是"游戏"，感受不到在"学习"
2. **即时反馈，寸劲爽感** — 每一次操作在200ms内有反馈，连击/击败/进化的仪式感拉满
3. **数据本地，零运维** — 不上云、不注册、不联网，一个人维护，所有人都能用
4. **3秒启动，随时放下** — 打开3秒内进入游戏，关闭不丢进度

### 2.4 产品边界（什么不做）

| 不做 | 原因 |
|------|------|
| 注册/登录系统 | 增加使用门槛，孩子不愿填表 |
| 好友聊天/社交功能 | 监管风险，不做社交平台 |
| 内购/付费墙 | 影响学习公平性 |
| 云端服务器 | 增加运维成本和隐私风险 |
| 多语言支持 | 当前目标用户只有中文母语者 |
| 教师端/家长后台 | 超出家庭使用场景 |
| 视频/直播课程 | 偏离游戏化学习定位 |

| G3 | 让孩子在游戏过程中实际积累英语词汇和语感 |
| G4 | 让孩子愿意开口说英语（口语模块） |

## 4. 用户画像与角色定义

### 4.1 核心用户：学生

#### 画像A：小学高年级（10-12岁）

| 维度 | 描述 |
|------|------|
| 基本信息 | 小学4-6年级，男女比例约6:4（男略多） |
| 英语水平 | KET级别 / 词汇量200-800 / 简单现在时和一般疑问句 |
| 设备 | 家长手机（限时使用）或自家iPad |
| 使用时段 | 晚饭后19:00-20:00、周末碎片时间 |
| 注意力特征 | 单次持续5-10分钟，容易被打断 |
| 心理特征 | 对"枯燥"零容忍，对"完成"有成就感，喜欢收集和展示 |
| 二次元接受度 | 极高。JOJO/鬼灭/咒术回战/宝可梦 都是日常话题 |
| 内在动机 | "我看JOJO！我也想要替身！" |
| 社交需求 | 在学校和同学聊"你打到哪个BOSS了？" |

**痛点：**
- 英语补习班已经很多了，再打开一个"学习APP"很抗拒
- 背单词看不到尽头，没有成就感
- 妈妈说的"玩中学"通常都是骗人的

#### 画像B：初中生（13-15岁）

| 维度 | 描述 |
|------|------|
| 基本信息 | 初一至初三，男女比例约5:5 |
| 英语水平 | PET级别 / 词汇量800-2000 / 基础语法已学 |
| 设备 | 自己的手机（大部分已有） |
| 使用时段 | 放学后17:00-18:00、睡前21:00-22:00 |
| 注意力特征 | 单次10-15分钟，有碎片化使用倾向 |
| 心理特征 | 考试压力大，需要"有用"的感觉，但也想放松 |
| 二次元接受度 | 高，但不像小学生那样外显 |
| 内在动机 | "中考要考" + "能放松一下" |
| 社交需求 | 不想被同学比下去，怕自己跟不上 |

**痛点：**
- 作业已经很多了，没有额外时间"学英语"
- 纯刷题太枯燥，但知道有用
- 口语最弱，不敢开口，怕丢脸

### 4.2 次要用户：家长

| 维度 | 描述 |
|------|------|
| 角色 | 孩子的父亲或母亲，30-45岁 |
| 核心需求 | 孩子能在玩的过程中**真的学到东西** |
| 决策因素 | 免费、无广告、数据安全、效果可见 |
| 操作能力 | 能把HTML文件发到手机上就算达标 |
| 担忧 | 怕孩子沉迷游戏、怕学不到东西、怕需要自己操作 |

**驱动因素：**
- 孩子主动说"我今天想玩那个英语游戏" → 家长惊喜
- 孩子无意中说出 game里学到的单词 → 家长放心

### 6.2 关键洞察

**多邻国的优势（可以借鉴）：**
- 每日打卡提醒机制非常成熟
- 碎片化学习（5分钟/次）设计得当
- 连击（Streak）的激励效应

**多邻国的劣势（我们要避开）：**
- UI太过卡通幼稚，初中生觉得"太小孩了"
- 学习内容偏通用，缺乏针对性
- 必须联网，数据在云端

**百词斩的优势：**
- 图片记忆法有效
- 词库体系完善

**百词斩的劣势（我们要避开）：**
- 游戏化程度低，本质上还是"背单词工具"
- 界面无聊，缺乏沉浸感

### 6.3 我们的差异化定位

```
         高游戏感
            │
    多邻国  │  ★ 我们
            │  (JOJO风格×替身收集)
            │
    低学习←─┼─→ 高学习
    感      │   感
            │  百词斩
            │
         低游戏感
```

我们占据的是**"高游戏感+中学习感"**这个象限，区别于多邻国的"中游戏感+中学习感"和百词斩的"低游戏感+高学习感"。

**核心差异点：**
1. **JOJO风格** — 二次元文化壁垒，竞品无法复制
2. **替身收集+进化** — 核心驱动力，不是"刷题"而是"变强"
3. **零门槛使用** — 无需安装注册，点开即用
4. **数据全本地** — 家长最放心的隐私方案

#### 9.1.3 解锁逻辑

```javascript
function regionAccessible(regionId, bossDone) {
  if (regionId === "animals") return true;
  if (regionId === "final") {
    return bossDone.includes("animals") && bossDone.includes("food")
      && bossDone.includes("school") && bossDone.includes("nature")
      && bossDone.includes("daily") && bossDone.includes("emotions");
  }
  const region = REGIONS.find(r => r.id === regionId);
  return bossDone.includes(region.prereq);
}
```

#### 9.1.4 区域内的对战流程（与现有BOSS战逻辑对接）

```
进入区域

  → 每次答对累计 regionEnergy++
  → regionEnergy >= 5 → 触发BOSS门开启动画
  → 进入BOSS战（复用现有startBoss逻辑）
  → 击败BOSS → 区域标记完成 → 解锁下一区域
```

```

    ↓

    ↓
```

### 11.3 数据安全

```javascript
// 读取（防御性）
function loadState(key, defaultValue) {
  try {
    const raw = localStorage.getItem(key);
    if (raw === null) return defaultValue;
    return JSON.parse(raw);
  } catch(e) {
    console.warn('Failed to load', key, ':', e);
    return defaultValue;
  }
}

// 写入（防御性）
function saveState(key, value) {
  try {
    localStorage.setItem(key, JSON.stringify(value));
  } catch(e) {
    console.warn('Failed to save', key, ':', e);
  }
}
```

### 12.1 设计语言

| 维度 | 规范 |
|------|------|

| 间距 | 基础 16px，大间距 24px，卡片内缩进 12-16px |
| 动效 | 即时反馈 200-400ms，过渡动画 400-800ms |
| 声效 | Web Audio API 合成（方波/锯齿波/正弦波） |

### 12.2 交互原则

1. **所见即所点** — 所有可交互元素必须有点击反馈（颜色变化/缩放/位移）
2. **即时反馈** — 任何操作在200ms内给出视觉或听觉反馈
3. **信息层级** — 主按钮 > 辅助操作 > 只读信息，视觉权重递减
4. **错误可逆** — 所有破坏性操作（重置）有二次确认
5. **进度可视** — 用户任何时候都知道"我在哪里"和"我完成了多少"

### 12.3 按钮规格

| 类型 | 尺寸 | 点击态 | 禁用态 |

| 选项按钮 | h:48px, 圆角12px | bg透明度提升 | opacity: 0.7 |

| 返回按钮 | 36px, 左对齐 | color变亮 | — |

### 12.4 加载与过渡状态

## 15. 风险评估与应对策略

### 15.1 风险矩阵

| 编号 | 风险描述 | 概率 | 影响 | 等级 | 应对策略 |
|------|---------|------|------|------|---------|
| R01 | 孩子觉得太难，放弃 | 中 | 高 | 🔴 | 初始难度降低，词库从KET开始 |
| R02 | 30个词太少，很快就背完 | 高 | 中 | 🔴 | Phase2扩展至90词，含变体题型 |
| R03 | BOSS打不过导致挫败感 | 中 | 高 | 🔴 | 3条命+无惩罚机制，鼓励重试 |
| R04 | 文件在企业微信传输失败（>100MB） | 低 | 高 | 🟡 | 当前文件54KB，远低于限制 |
| R05 | iOS Safari对file://有限制 | 中 | 中 | 🟡 | 提示使用"打开文件"功能 |
| R06 | 孩子在其他设备上进度不同步 | 高 | 中 | 🟡 | Phase3做iCloud同步，当前先手动复制localStorage |
| R07 | 孩子不喜欢JOJO风格 | 低 | 高 | 🟢 | 产品假设就是JOJO粉丝，不做下沉 |
| R08 | 家长觉得"只是游戏，没学到东西" | 中 | 中 | 🟢 | 个人中心的统计数据就是给家长看的 |
| R09 | 连续打卡断了，孩子失落放弃 | 中 | 中 | 🟢 | 倍率从3天开始，短周期给正反馈 |
| R10 | 孩子在学校炫耀导致其他同学也想用 | 低 | 正面 | 🟢 | 是企业微信传播的优势场景 |

### 15.2 降级策略

| 功能 | 降级方案 | 何时触发 |
|------|---------|---------|
| 探索地图 | 回退到线性BOSS列表（Phase0模式） | 地图实现遇到设计问题 |
